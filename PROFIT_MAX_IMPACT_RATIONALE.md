# PROFIT_MAX_IMPACT_RATIONALE

각 변경이 기대수익 개선 또는 실전 괴리 축소에 기여하는 이유를 요약합니다.

## 1) StrategyEngine 단일화
- 추천/자동매수/자동매매가 동일 엔진 신호를 사용하면 경로별 드리프트가 줄어듭니다.
- 동일 입력에서 동일 의사결정 경로를 강제해 운영 중 재현성을 높입니다.

## 2) EV 기반 랭킹
- 점수 중심 랭킹보다 `win_probability x upside - lose_probability x downside` 구조가 직접적인 기대수익 의사결정에 가깝습니다.
- `expected_value <= 0` 진입 금지로 음의 기대값 포지션 진입을 차단합니다.

## 3) 통계 캘리브레이션 + shrinkage
- 표본 수가 적은 셀의 과최적화를 줄이고, 신뢰도(`reliability`)를 수치와 함께 노출합니다.
- 검증 거래수/샤프가 낮을수록 prior 쪽으로 수축해 실전 성과 변동을 완화합니다.

## 4) Risk-based sizing
- 균등배분 대비 손절거리 기반 사이징은 동일 손실 예산 내 종목 간 리스크를 정규화합니다.
- EV 강도/신뢰도/노출 캡을 결합해 높은 확신 신호에 자본을 효율 배분하고, 과집중을 방지합니다.

## 5) 실행 현실화(슬리피지/유동성/갭)
- 장초/변동성/거래량 기반 슬리피지 반영으로 백테스트 과대성과를 줄입니다.
- ADV 비율 제한과 최소 거래량 게이트로 비현실적 주문 체결을 차단합니다.
- 손절/청산 시 갭 리스크 보정 체결가를 적용해 실전 손실 분포를 더 보수적으로 반영합니다.

## 6) Regime/Risk Guard
- risk-off/손실한도/연속손실 쿨다운은 전략 생존성을 우선 보장합니다.
- 수익률 극대화 이전에 파산확률 관리가 가능해 장기 복리 경로를 보호합니다.

## 7) Validation 고도화
- 단일 백테스트 수치보다 walk-forward OOS 결과를 함께 노출해 일반화 성능 검증이 가능합니다.
- 전략/구간별 지표 분해(Profit Factor, MDD, OOS reliability)로 과최적화 징후를 조기 식별합니다.
