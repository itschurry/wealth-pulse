# daily-market-brief 웹 UI 사용 매뉴얼

이 문서는 웹 콘솔에서 **각 화면이 무슨 용도인지**, **버튼이 정확히 무슨 동작을 하는지**를 운영자 관점에서 정리한 매뉴얼입니다.

대상 URL:

- 개발: `http://127.0.0.1:5173`
- Docker 배포: `http://127.0.0.1:8081`

---

## 1. UI 전체 구조

앱은 크게 2개 상위 섹션으로 나뉩니다.

### 1-1. 상단 1차 탭

- **콘솔**
  - 운영 상태 확인, 엔진 제어, 검증 실행용
- **리포트**
  - 오늘 판단, 실행 우선순위, 관망/관심 포인트 브리핑용

즉,

- **뭔가 실행/제어할 때** → 콘솔
- **읽고 판단할 때** → 리포트

---

## 2. 공통 UI 요소

대부분 화면 상단에는 공통 액션바가 있습니다.

### 2-1. `새로고침`

의미:

- 현재 화면 데이터를 다시 불러옴
- API 재호출
- 화면 캐시 상태 점검

언제 쓰나:

- 방금 백엔드 작업이 끝났을 때
- 엔진 상태가 바뀌었을 것 같을 때
- 화면 일부만 비어 있을 때

### 2-2. `로그 보기`

의미:

- 현재 화면에서 쌓인 실행 로그를 우측 드로어로 엶
- 정보/성공/경고/오류 로그 필터링 가능

안에서 가능한 것:

- 검색
- 레벨 필터 (`전체`, `정보`, `성공`, `경고`, `오류`)
- 작업 소스 필터 (`백테스트`, `최적화`, `설정`, `모의투자` 등)
- `로그 비우기`

### 2-3. `설정`

의미:

- 화면별 설정 패널을 우측 드로어로 엶
- 화면마다 다른 설정이 들어감

특징:

- `저장 필요` 배지가 보이면 아직 화면 내 draft가 저장되지 않은 상태
- 일부 설정은 **브라우저 localStorage 기준**으로 저장됨

### 2-4. 상태 카드(상단 요약 칩)

각 페이지 상단에 있는 작은 상태 카드들은 단순 장식이 아니고, 해당 화면에서 **먼저 볼 핵심 숫자**만 요약합니다.

예:

- 엔진 상태
- 오늘 주문 수
- OOS 신뢰도
- 차단된 신호 수

## 4. 콘솔 > 개요 화면

경로:

- `/console/overview`

역할:

- 엔진, 손익, 차단 상태를 **한눈에** 보는 대시보드
- 제어보다 **상태 파악**이 우선인 화면

스크린샷:

![개요 대시보드](./assets/screenshots/overview.png)

### 4-1. 상단 카드 의미

- **엔진 상태**
  - 실행 중 / 일시정지 / 중지 / 오류
- **다음 실행**
  - 엔진이 다음 루프를 도는 예상 시각
- **오늘 주문(B/S)**
  - Buy / Sell 체결 건수
- **오늘 실현손익**
  - 오늘 확정 손익

### 4-2. 본문 카드 의미

- **최근 실행 시각**
  - 엔진이 마지막으로 돈 시각
- **현재 Equity**
  - 현재 총자산 기준치
- **포지션 수**
  - 현재 보유 종목 수
- **오늘 실패/스킵**
  - 실패 주문 + 스킵된 실행 합계
- **Validation Gate**
  - 검증 정책이 활성인지 여부
- **Optimized Params**
  - 최적화 파라미터가 stale인지 active인지

### 4-3. 리스크 가드 박스 의미

- **현재 진입 제한**
  - 신규 진입을 지금 허용하는지
- **일일 손실 잔여**
  - 손실 한도까지 남은 폭
- **사유**
  - 왜 차단됐는지 이유 목록

### 4-4. 이 화면에서 해야 하는 것

- 엔진이 돌고 있는지 확인
- stale optimization 경고 있는지 확인
- 오늘 실패/스킵 급증 여부 확인
- 신규 진입 제한이 켜져 있으면 이유 먼저 확인

이 화면은 **“지금 시스템이 정상인가?”** 를 제일 빨리 답하는 용도야.

## 5. 콘솔 > 신호 화면

경로:

- `/console/signals`

역할:

- 종목별 진입 허용/차단 이유를 표로 점검
- 추천 후보와 차단 사유를 운영 기준으로 확인

### 5-1. 상단 상태 요약 의미

- **전체 신호**: 현재 표시 중인 신호 수
- **진입 허용**: 실제로 들어갈 수 있는 신호 수
- **차단**: 조건 때문에 막힌 신호 수
- **장세/위험도**: 현재 시장 레짐과 리스크 수준

### 5-2. 표 컬럼 의미

- **종목**: 코드/이름/시장
- **전략**: 돌파, 눌림목, 이벤트 등
- **점수/신뢰**: 점수와 승률/신뢰 지표
- **EV**: 기대값
- **진입**: 추천 / 차단
- **권장 수량**: 권장 주문 수량
- **검증 신뢰도**: 높음/보통/낮음/부족
- **유동성/슬리피지**: 체결 현실성 관련 지표
- **사유**: 차단 사유 또는 판단 근거

### 5-3. 표 내부 인터랙션

- **사유 > 상세 N건**
  - 눌러서 reason code 목록을 펼침

### 5-4. 이 화면에서 해야 하는 것

- 허용 신호 수가 급감했는지 보기
- 차단 사유가 validation / liquidity / risk-off 때문인지 확인
- EV는 높아도 신뢰도가 낮은 후보는 따로 걸러보기

## 6. 콘솔 > 모의투자 화면

경로:

- `/console/paper`

역할:

- 모의 계좌 상태 확인
- 엔진 시작/정지/일시정지/재개
- 위험 포지션 점검
- 주문/사이클 로그 확인

스크린샷:

![모의투자 운영 화면](./assets/screenshots/paper.png)

이 화면은 사실상 **운영실**이야.

### 6-1. 상단 액션 버튼 의미

#### `엔진 시작`

의미:

- 자동 실행 루프 시작
- 선택한 시장 기준으로 신호 평가 시작

실행 전 확인할 것:

- 설정에서 시장이 최소 1개 체크됐는지
- interval, max positions 같은 값이 맞는지

주의:

- 이미 실행 중이면 비활성화됨
- 확인 팝업이 뜸

#### `일시정지`

의미:

- 현재 포지션은 유지
- 신규 자동 실행만 잠깐 멈춤

언제 쓰나:

- 장중 변동성이 너무 클 때
- 잠시 신규 평가를 멈추고 싶을 때

#### `재개`

의미:

- 일시정지 상태에서 자동 실행 재시작

#### `엔진 중지`

의미:

- 자동 실행 루프 완전 중지
- 신규 평가와 자동 집행 중단

주의:

- 확인 팝업 있음
- 운영 중지 성격이라 일시정지보다 강함

#### `강제 새로고침`

의미:

- 콘솔 스냅샷 + 계좌 + 엔진 상태 + 런타임 로그를 전부 다시 불러옴

언제 쓰나:

- 방금 상태가 바뀌었는데 화면 반영이 늦을 때
- order/cycle 로그 갱신 확인할 때

#### `모의투자 초기화`

의미:

- 계좌, 포지션, 기준 자금, 일부 로그 기준점 초기화

주의:

- 되돌릴 수 없음
- 확인 팝업 있음
- 테스트 리셋용이지 운영 중에는 함부로 누르면 안 됨

### 6-2. 설정 드로어 항목 의미

`설정` 버튼을 누르면 나오는 항목들:

- **초기 원화 현금**
  - 리셋 시 사용할 KRW 시작 자금
- **초기 달러 현금**
  - 리셋 시 사용할 USD 시작 자금
- **모의투자 기간(일)**
  - 리셋/기준 시뮬레이션 기간
- **시장 선택 (KOSPI / NASDAQ)**
  - 엔진 시작 대상 시장
- **최대 포지션 수(건)**
  - 시장당 보유 포지션 상한
- **일일 매수 제한(건)**
  - 하루 매수 횟수 제한
- **일일 매도 제한(건)**
  - 하루 매도 횟수 제한
- **종목당 일일 주문 제한(건)**
  - 같은 종목 재주문 제한
- **설정 저장**
  - 위 값을 브라우저 localStorage 에 저장

중요:

- 설정 저장은 **백엔드 영구 설정 파일 수정이 아니라 UI 저장값 기준**이야
- 실제 엔진 시작 시 이 저장된 값을 읽어 요청 payload로 씀

### 6-3. 본문 섹션 의미

#### `운용 우선순위`

- 리스크 정리 먼저 / 정상 운영 지속 같은 오늘의 운영 방향
- 위험 포지션, 실패 주문, 엔진 신뢰도를 먼저 보게 만든 요약 블록

#### `운용 긴급도`

- 신규 진입 허용 여부
- 위험 포지션 수
- 실패 주문 수
- 다음 실행 시각

#### `현금/자산 상태`

- 총자산
- 원화/달러 현금
- 평가손익 / 실현손익

#### `우선 확인할 위험 포지션`

- 손실이 심하거나 오래 묶인 포지션 상위
- 먼저 정리/관찰할 대상

#### 포지션 테이블

컬럼 의미:

- 종목 / 시장 / 수량
- 진입가 / 현재가
- 평가손익 / 수익률
- 보유기간
- 손절가/손절률
- 익절가/익절률
- 전략 태그
- 상태

#### `엔진 상태 패널`

여기서 꼭 볼 것:

- 최근 오류
- 다음 실행 시각
- today 주문(B/S/F)
- validation gate 활성 여부

#### `최근 체결 내역`

- 최근 주문 성공/실패 이력
- 실패 시 사유 확인 가능

#### `최근 엔진 이벤트 로그`

- skip reason count
- cycle별 매수/매도/오류

### 6-4. 이 화면에서의 운영 루틴

권장 순서:

1. 엔진 상태 확인
2. 신규 진입 허용 여부 확인
3. 위험 포지션 확인
4. 오늘 실패 주문 확인
5. 필요 시 일시정지/재개/중지

## 7. 콘솔 > 백테스트/검증 화면

경로:

- `/console/validation`

역할:

- 검증 설정 관리
- 백테스트 실행
- 최적화 실행
- 채택/거절 판단 근거 확인

### 7-1. 설정 드로어 항목 의미

#### `시장`
- 백테스트 대상 시장 선택 (`KOSPI` / `NASDAQ`)

#### `전략 이름`
- 실행 이력과 저장 이력에 함께 남는 라벨

#### `백테스트 기간(일)`
- 전체 회고 기간
- 길수록 느려질 수 있지만 검증 안정성은 좋아짐

#### `학습 기간(일)`
- train 구간 길이

#### `검증 기간(일)`
- validation 구간 길이

#### `Walk-forward`
- 구간별 재학습 사용 여부

#### `최소 거래 수(건)`
- 검증 통과 최소 거래 수 기준

#### `목표 함수`
- 최적화 판단 기준
- 수익 우선 / 수익+안정 균형

#### `설정 저장`
- 현재 draft를 저장 상태로 반영
- 상단의 `저장 필요` 배지 제거

#### `초안 초기화`
- draft를 기본값으로 되돌림
- 저장 안 한 값은 사라짐
- 확인 팝업 있음

### 7-2. 실행 패널 버튼 의미

#### `백테스트 실행`

의미:

- 현재 저장/초안 설정 기준으로 백테스트 실행
- 완료 후 성과 요약과 이력 갱신

언제 쓰나:

- 파라미터 바꾼 직후
- 최근 전략 상태 다시 보고 싶을 때

#### `최적화 실행`

의미:

- 백그라운드 최적화 요청
- 실행 중이면 중복 실행 대신 기존 작업 추적

언제 쓰나:

- 채택 전 파라미터 개선 시도
- stale optimization 경고가 있을 때

### 7-3. 주요 카드 의미

#### `전략 채택 상태`

값 예시:

- 즉시 사용 가능
- 보류
- 거절

의미:

- OOS 수익률, PF, 낙폭, 거래 수, 신뢰도를 합쳐 운영 반영 판단을 제시

#### `전략 점수카드`

- 왜 점수가 높거나 낮은지 분해해서 보여줌
- 점수 하나만 보지 않게 하기 위한 카드

#### `테일 리스크 가드레일`

- worst loss
- expected shortfall
- 하위 5% 손실 경계

이 영역은 **“수익이 나더라도 꼬리가 위험한지”** 확인하는 용도야.

#### `실행 패널`

- 현재 저장된 설정 요약 표시
- `저장 필요`면 실행 전 저장 권장
- 백테스트/최적화 현재 phase 표시

#### `참고 이력 / 로그`

- 실행 이력
- 최적화 이력
- 저장 이력
- 오류 로그
- 사유별 성과

### 7-4. 이 화면에서 권장하는 사용 순서

1. 설정 수정
2. `설정 저장`
3. `백테스트 실행`
4. 결과 보고 부족하면 `최적화 실행`
5. OOS / 낙폭 / 신뢰도 보고 채택 여부 결정

5. 근거 요약

### 8-2. 주요 카드 의미

- **오늘 결론**
  - 방어 / 중립 / 선별 / 공격 같은 당일 운영 모드
- **허용 후보 평균 점수**
  - 허용 가능한 종목들의 평균 scorecard 수준
- **오늘 1순위 전략 상태**
  - 가장 우선순위 높은 후보의 상태
- **테일리스크 경고**
  - 꼬리 손실 관점에서 위험한 후보 수
- **오늘 실행**
  - 바로 해야 할 액션 리스트
- **오늘 금지/회피**
  - 피해야 할 행동 요약
- **근거 요약**
  - 뒤늦게 참고하는 보조 근거

### 8-3. 버튼

#### `새로고침`
- 리포트 관련 데이터 다시 조회

이 화면은 버튼이 많지 않아.
핵심은 **읽는 순서를 강제하는 정보 구조**야.

## 9. 리포트 > 운영알림

경로:

- `/reports/alerts`

역할:

- 액션보드 대신 **실제 운영에 영향을 주는 경고만 먼저 모아보는 화면**
- 읽고 끝내는 요약이 아니라 **지금 조치가 필요한 문제를 빠르게 잡는 화면**

### 9-1. 상단 카드 의미

- **엔진 상태**
  - 실행 중 / 일시정지 / 중지 / 오류
  - 최근 오류가 있으면 상세 문구가 같이 보임
- **신규 진입 제한**
  - 리스크 가드가 신규 진입을 막고 있는지 여부
  - 막고 있으면 사유가 함께 표시됨
- **최적화 파라미터**
  - stale 여부 확인
  - stale이면 검증 화면에서 최적화 재실행 우선
- **실패/스킵**
  - 오늘 실패 주문 수와 최근 스킵 수
- **Validation Gate**
  - validation gate 활성 여부
  - 현재 OOS 신뢰도와 최소 거래 수 기준 같이 표시
- **알림 채널**
  - 텔레그램 등 알림 채널이 실제로 설정 완료인지 확인

### 9-2. `지금 바로 할 일` 카드 의미

이 영역은 체크리스트가 아니라 **조치 우선순위 카드**야.

예:

- 엔진이 멈춰 있으면 시작 여부 판단
- 리스크 가드 차단 사유 먼저 확인
- stale optimization이면 재실행
- 실패 주문 있으면 최근 체결/이벤트 로그 확인
- 알림 채널 설정 미완료면 운영 전에 먼저 수정

### 9-3. `운영 기준값` 카드 의미

- 허용 / 차단 신호 수
- 다음 실행 시각
- 최근 성공 시각
- 오늘 실현손익
- 일일 손실 잔여

이건 지금 상태를 빠르게 읽는 운영용 숫자판이라고 보면 됨.

### 9-4. 하단 리스트 의미

#### `리스크 가드 사유`
- 신규 진입 차단 이유를 한 줄씩 보여줌
- 왜 막혔는지 먼저 읽는 용도

#### `최근 차단 신호 예시`
- 최근 차단된 종목과 차단 이유 예시
- 어떤 종목이 왜 빠졌는지 빠르게 보는 용도

#### `알림 채널 상태`
- channel / enabled / configured / chat_id_configured / last_error
- 운영 중 메시지 누락 가능성 점검용

### 9-5. 언제 먼저 보나

- 엔진이 왜 안 도는지 모르겠을 때
- 오늘 신규 진입이 왜 막혔는지 볼 때
- stale optimization 경고가 떴을 때
- 실패 주문이 늘었을 때
- 텔레그램 알림이 안 오는 것 같을 때

## 10. 리포트 > 관망/관심목표

경로:

- `/reports/watch-decision`

역할:

- 오늘 신규 진입 태도와 관심 포인트를 짧게 보는 화면

### 10-1. 주요 카드 의미

- **신규 진입**: 가능 / 제한
- **위험도**: 현재 risk level
- **허용 신호**: 오늘 실제 진입 후보 수
- **판단 근거**: 관망/집중 이유 요약

이 화면은 짧은 판단 카드라서,
상세 제어보다 **지금 들어갈지 말지** 빠르게 보는 용도야.

## 12. 운영자 추천 사용 순서

### 아침 브리핑 루틴

1. `리포트 > 오늘`
2. `리포트 > 운영알림`
3. `리포트 > 관망/관심목표`
4. `콘솔 > 개요`
5. 필요 시 `콘솔 > 신호` / `콘솔 > 모의투자`

### 엔진 운영 루틴

1. `콘솔 > 개요` 에서 상태 확인
2. `콘솔 > 모의투자` 에서 위험 포지션/실패 주문 확인
3. 필요 시 `일시정지` 또는 `재개`
4. 큰 변경 전에는 `강제 새로고침`

### 전략 점검 루틴

1. `콘솔 > 백테스트/검증`
2. 설정 확인
3. `설정 저장`
4. `백테스트 실행`
5. 필요 시 `최적화 실행`
